# Public Build (No Source)

This folder contains only production build files from the private source project.

## Publish this folder to GitHub
1. Create a new public repository on GitHub (for example: repair-pos-showcase).
2. Upload everything inside this folder to that repository root.
3. Enable GitHub Pages from the repository settings.

## Important
- This excludes the original source files.
- Build assets are still client-side files, so they can be inspected in the browser.
- Source maps are disabled to reduce source visibility.
